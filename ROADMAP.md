# GEO-Master 90 天路线图

目标：用案例、模板、可复现实验和稳定更新节奏，把 GEO-Master 做成中文出海 GEO 领域最实用的开源项目之一。

## 第 1 阶段：建立可信度（第 1–2 周）

- [x] 完成 README 与项目定位
- [x] 建立案例驱动的内容结构
- [ ] 发布 Case 001：实验室仪器 Reddit GEO 拆解
- [ ] 发布案例证据评级标准
- [ ] 发布统一 CASE 模板
- [ ] 建立贡献指南与案例投稿入口
- [ ] 完成首批 20 个 Issue，公开内容计划

首批文章：

1. GEO 案例应该如何验证，而不是只看截图
2. 实验室仪器 Reddit GEO：136 单案例拆解
3. 为什么 Ahrefs 有外链不等于 AI 会推荐
4. Reddit GEO 的合规方法与封号风险
5. 如何测量 AI 提及、引用、访问和订单

## 第 2 阶段：提供可直接使用的模板（第 3–4 周）

- [ ] GEO 可见性基线问题集
- [ ] 每周监测表
- [ ] 采购问题库模板
- [ ] 内容 Brief 模板
- [ ] Reddit 回复检查清单
- [ ] GEO 询盘与订单归因表
- [ ] 案例证据矩阵模板

## 第 3 阶段：形成案例矩阵（第 2 个月）

计划覆盖：

- [ ] B2B 实验室仪器
- [ ] 户外电源与充电产品
- [ ] 庭院机器人
- [ ] 智能安防
- [ ] 跨境 SaaS
- [ ] 新品牌实体冷启动
- [ ] 产品参数更新与 AI 旧数据问题
- [ ] GEO 提及上涨但没有转化的失败案例

每个案例至少包含：

- 业务背景；
- 原始宣称；
- 完整实施过程；
- 技术机制解释；
- 数据与证据核查；
- 可复制动作；
- 不可复制或不合规动作；
- 复现实验计划。

## 第 4 阶段：发布自有实验（第 3 个月）

- [ ] 选择一个真实品牌建立基线
- [ ] 创建 50 个高价值查询
- [ ] 记录不同 AI 平台的原始回答
- [ ] 改造产品页、FAQ 与对比内容
- [ ] 连续监测 30 天
- [ ] 发布完整数据与失败结果
- [ ] 建立第一个 GEO Benchmark 数据集

## 快速涨星节奏

### 每周固定节奏

- 周一：发布一个案例或案例更新
- 周三：发布一份可下载模板
- 周五：发布一个复现实验或失败复盘

### 每月固定产物

- GEO 平台变化月报
- 新增案例索引
- Top 10 高价值问题与来源变化
- 开源贡献者感谢名单

## 里程碑

### 100 Stars

- 完成 5 个完整 CASE
- 完成 5 个可直接使用的模板
- 发布第一次 30 天监测报告

### 500 Stars

- 完成 15 个案例
- 发布 GEO Benchmark v0.1
- 增加英文版 README
- 引入外部案例投稿

### 1,000 Stars

- 发布 GEO-Master CLI 或轻量审计工具
- 建立公开案例数据库
- 提供自动化周报与趋势监测
- 与出海品牌共同发布可验证案例

## 内容原则

- 不追求每天更新低价值内容；
- 优先发布别人能 fork、复制和复现的材料；
- 所有结果数字标记证据等级；
- 失败结果同样公开；
- 不用伪装用户、批量刷帖等方式换取短期效果。
