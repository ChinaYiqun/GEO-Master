# 参与 GEO-Master

感谢你愿意补充案例、数据、工具或质疑。这个项目优先接受**可验证、可复现、能落地**的内容。

## 可以贡献什么

- 中国品牌出海 GEO 案例；
- ChatGPT、Perplexity、Gemini、Claude、Google AI Search 的复现实验；
- 失败案例、封号经历、错误归因和无效优化；
- 查询集、监测表、Prompt、内容 Brief；
- 平台官方文档和规则变化；
- 对已有案例证据链的补充或纠错。

## 提交案例前

1. 复制 [CASE-TEMPLATE.md](CASE-TEMPLATE.md)；
2. 标记案例类型：Verified、Claimed、Teardown、Replication 或 Failure；
3. 将事实、案例方宣称和个人判断分开；
4. 给每个关键数字注明来源、时间范围和统计口径；
5. 删除客户隐私、账号凭证和未授权商业数据；
6. 说明利益关系，例如作者是否为服务商、品牌方或工具方。

## 不接受的内容

- 伪装普通用户、隐藏品牌关系的营销教程；
- 批量刷帖、虚构评论或操纵社区讨论；
- 没有来源却写成确定事实的订单和增长数字；
- 只展示一次有利 AI 回答的“成功案例”；
- 大量改写网络文章但没有新增分析；
- 纯工具推广且不披露利益关系。

## 推荐的目录命名

```text
cases/<industry>/case-<number>-<short-name>/README.md
playbooks/<topic>.md
explainers/<topic>.md
templates/<name>.md
failures/<topic>.md
```

## Pull Request 检查清单

- [ ] 标题能说明行业、渠道和问题；
- [ ] 已注明案例类型和证据等级；
- [ ] 所有关键数据都有来源；
- [ ] 已说明无法验证的部分；
- [ ] 已加入失败模式和不可复制部分；
- [ ] 外部内容以总结和引用为主，没有大段搬运；
- [ ] 没有泄露隐私和商业机密；
- [ ] Markdown 链接可以访问。

## 适合新贡献者的任务

可以从标有 `good first issue` 的任务开始：

- 核验一个公开 GEO 案例；
- 补充一个平台官方来源；
- 制作一份查询集；
- 增加失败案例；
- 修复链接、格式或错别字。

讨论有争议的结论时，请优先提供原始来源、可重复步骤和反例。
