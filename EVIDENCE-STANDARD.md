# GEO-Master 证据与案例评级标准

GEO 行业中常见大量截图、单次回答、服务商自报数据和无法追溯的订单数字。为了避免把营销材料误写成确定事实，GEO-Master 对每条重要结论进行证据分级。

## 证据等级

| 等级 | 证据类型 | 示例 |
|---|---|---|
| A | 官方文档、公开原始数据、可公开查询记录 | 平台官方说明、公开日志、可访问页面 |
| B | 企业后台、CRM、分析平台、完整连续截图 | GA4、Search Console、CRM 导出 |
| C | 可以重复执行的黑盒实验 | 同一问题多平台、多轮次测试 |
| D | 案例方单方面陈述 | 服务商文章、采访中的结果数字 |
| E | 二手转述、无法找到原始出处 | 汇总文章、社交媒体转述 |
| X | 已发现矛盾、无法成立或明显误导 | 工具能力与结论不匹配 |

## 案例类型

### Verified Case

至少满足：

- 企业或品牌可确认；
- 实施动作可确认；
- 有实施前后数据；
- 有原始截图、后台数据或公开记录；
- 说明转化归因方法；
- 能排除明显替代解释。

### Claimed Case

案例方宣称获得某种结果，但缺乏足够材料独立验证。

标准写法：

> 案例方宣称获得该结果，公开材料暂不足以独立验证。

### Teardown

基于公开材料拆解案例，重点分析：

- 做了什么；
- 哪些环节可能有效；
- 哪些属于推测；
- 哪些存在合规或归因问题；
- 如何设计复现实验。

### Replication

GEO-Master 自主执行并公开过程与数据的复现实验。

### Failure Case

公开没有达到目标、出现副作用或无法归因的案例。

## 结果状态字段

每个案例建议在顶部声明：

```yaml
case_type: teardown
result_status: claimed
independent_verification: unavailable
attribution_confidence: low
reproducibility: medium
compliance_risk: medium
last_verified: 2026-07-17
```

## 证据矩阵模板

| 声明 | 来源 | 是否可复核 | 证据等级 | 当前判断 |
|---|---|---|---|---|
| 品牌提及率从 0 提升到 35% | 服务商文章 | 部分 | D | 案例方宣称 |
| Perplexity 引用了官网 | 原始回答链接或连续截图 | 是 | B/C | 可确认或可复现 |
| 获得 136 笔 GEO 订单 | 案例文章 | 否 | D | 无法独立归因 |
| Ahrefs 显示新增外链 | Ahrefs 导出 | 是 | B | 只能证明外链变化 |

## 订单归因最低要求

若案例宣称 GEO 带来订单，至少说明：

- 时间范围；
- 订单总量和增量；
- 首次接触来源；
- 最终转化来源；
- UTM、referral、品牌词或客户自报数据；
- 是否排除广告、SEO、老客户和销售跟进；
- 使用何种归因模型。

否则只能写成“相关订单”或“案例方宣称”，不能写成已证明的 GEO 贡献。

## AI 回答测试最低要求

- 每个问题至少运行 5 次；
- 记录平台、日期、地区、语言和登录状态；
- 保存完整回答和引用链接；
- 区分新对话与多轮对话；
- 不用单次回答代表平台长期行为；
- 平台变化后重新验证。

## 工具边界

- Ahrefs：外链、关键词和搜索可见性，不证明 AI 推荐；
- Search Console：Google 搜索表现，不等同于所有 AI 平台引用；
- GA4：访问行为，不自动证明访问由某个 AI 回答导致；
- Perplexity/ChatGPT 截图：证明一次结果，不证明稳定性；
- CRM：可支持商业归因，但取决于来源字段和录入质量。

## 修改与质疑

任何人都可以通过 Issue 或 Pull Request：

- 补充原始证据；
- 纠正错误归因；
- 提交相反实验结果；
- 将案例等级上调或下调。

证据等级不是永久结论，会随新增材料更新。
